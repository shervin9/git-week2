Interactive Rebase در Git

مدیریت تاریخچه‌ی گیت به صورت مؤثرتر!

تعریف Interactive Rebase

Interactive Rebase یک قابلیت در سیستم کنترل نسخه‌ی گیت است که به شما امکان می‌دهد تغییرات ثبت‌شده (commits) را ویرایش، حذف یا ترکیب کنید. این ویژگی به شما اجازه می‌دهد تا تاریخچه‌ی پروژه‌ی خود را مرتب‌تر و معنادارتر کنید.

چرا از Interactive Rebase استفاده کنیم؟

  • تنظیم تاریخچه: با استفاده از Interactive Rebase می‌توانید تاریخچه‌ی commits را به گونه‌ای ساماندهی کنید که مفهوم بهتری از تغییرات ایجاد شده در پروژه ارائه دهد.
  • ترکیب commits: این قابلیت به شما اجازه می‌دهد تا چندین commit مرتبط را ترکیب کنید و به یک commit تغییر دهید، که می‌تواند تاریخچه را تمیزتر کند.
  • ویرایش commits: می‌توانید پیام commit را ویرایش کنید یا اطلاعات اضافه‌ای به commits قدیمی بیفزایید.

نحوه‌ی استفاده از Interactive Rebase

  1. ورود به دستور rebase: 
    git rebase -i HEAD~n
    در این جا n تعداد commits اخیر است که می‌خواهید بر روی آن‌ها کار کنید.
  2. یک ویرایشگر متن باز می‌شود، جایی که می‌توانید عمل‌کردهای مختلف مانند pick، squash، و edit را برای commits مشخص کنید.
  3. سپس تغییرات را ذخیره کنید و به گیت بگویید که فرآیند rebase را ادامه دهد.
  4. در نهایت، اگر هیچ تعارضی وجود نداشته باشد، فرآیند به پایان می‌رسد.

مزایای استفاده از Interactive Rebase

  • ایجاد تاریخچه‌ای واضح و معنادار.
  • بهینه‌سازی مراحل توسعه.
  • مدیریت تعارضات به‌صورت مؤثر.

برای اطلاعات بیشتر، به مستندات گیت مراجعه کنید.
