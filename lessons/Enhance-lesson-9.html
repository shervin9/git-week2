Lesson 9: Cherry-Picking

Learn how to create and switch between branches

What is Cherry-Picking in Git?

Cherry-picking in Git is the process of selecting a specific commit from one branch and applying it to another branch. Unlike merging or rebasing, which typically involve integrating multiple commits or entire branches, cherry-picking allows you to apply individual commits selectively. This is useful when you want to incorporate a particular change-such as a bug fix or a feature update-without merging all the changes from the source branch.

Did you know? You can cherry-pick multiple commits at once using a range: git cherry-pick A^..B

How to Cherry-Pick a Commit

The command used is:

git cherry-pick <commit-hash>

Here, <commit-hash> is the unique identifier of the commit you want to apply. When you run this command on your current branch, Git copies the changes from the specified commit and creates a new commit on your branch with those changes, but with a new commit hash. This keeps the commit history clean and precise.

Example Scenario

Suppose you fixed a bug on the feature/login-bugfix branch and want to apply that fix to main without merging other changes:

git checkout main
git cherry-pick a1b2c3d4

Common Use Cases for Cherry-Picking

  • Backporting bug fixes from a development branch to a stable or release branch.
  • Applying hotfixes to production branches without merging ongoing development.
  • Correcting commits accidentally made to the wrong branch.
  • Reusing specific changes across multiple branches without merging entire branches.

Cherry-Picking: Pros and Cons

Pros Cons
Granular control over changes Can cause duplicate commits if not careful
Avoids merging unwanted changes May lead to conflicts
Quick Quiz:

Which command applies a specific commit from another branch to your current branch?

  • A. git merge
  • B. git cherry-pick
  • C. git rebase

Answer: B

Cherry-picking can be done for single commits, multiple commits, or a range of commits, and it can also be applied to commits from remote branches by fetching them first.

In graphical Git clients like GitHub Desktop or GitKraken, cherry-picking is supported with easy UI options to select commits and apply them to other branches without using the command line.

Summary

In summary, cherry-picking is a precise way to move individual commits between branches, giving developers granular control over which changes to integrate where, without merging whole branches or histories.
